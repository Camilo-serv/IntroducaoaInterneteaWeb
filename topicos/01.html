Visão Geral da Internet  

A Internet é, basicamente, uma rede de redes de computadores, ou seja, um conjunto de tecnologias de comunicação e protocolos que permitem a diferentes computadores ligados em diferentes redes, se comunicarem entre si. Um bom exemplo, muito usual nos dias atuais, é a comunicação entre um smartphone e um notebook, estando o smartphone numa rede móvel de pacotes (UMTS, HSPA e LTE) e o notebook numa rede local Wireless Ethernet.

Se a Internet é uma tecnologia que permite a intercomunicação entre Redes de Computadores, é preciso saber o que é uma Rede de Computadores. Você sabe? Veja este vídeo para entender o que são essas redes:

https://www.youtube.com/watch?v=MSiripzIJtg

A Internet funciona, principalmente, com o protocolo Internet Protocol (IP), que serve de base para a divisão da informação a ser transmitida em pacotes que poderão ser dirigidos até um computador destino, através de equipamentos que estabelecem rotas para os pacotes. Estes equipamentos são chamados de Roteadores.

Os pacotes que trafegam na internet usando o protocolo IP podem sofrer atrasos em sua chegada a um destino, além de retardos nos Roteadores e outros equipamentos de rede. Isso causa problemas para as aplicações que estejam usando este protocolo. Creio que você já deva ter visto estes problemas na forma de vídeos que demoram a ser exibidos, páginas da Web que demoram a ser exibidas. Além destes problemas de retardo, existem as perdas de pacote que são geradas por sobrecargas nos Roteadores ou nas filas de espera de equipamentos de redes. A perda de pacotes acaba gerando falhas na comunicação das aplicações que estão usando a internet.

Introdução ao TCP/IP

Vimos até aqui como a internet funciona, em linhas gerais, bem como sua história. Creio que você tenha percebido que o nome de dois protocolos foram muito citados no material visto. São eles o Internet Protocol (IP) e o Transmission Control Protocol (TCP). Saber como funcionam estes protocolos é importante para detectar problemas nas aplicações que estão sendo executadas na internet. Para quem trabalha ou quer desenvolver aplicações ou sites web, entender estes protocolos pode permitir a identificação de problemas futuros na aplicação.

A pilha de protocolos TCP/IP é vital para o funcionamento da internet. No entanto, aplicações que necessitem de grande e contínuo fluxo de pacotes demandam um protocolo mais simples e rápido que o TCP. Neste cenário, recomenda-se a adoção do User Datagram Protocol (UDP).

Existem outros protocolos na rede internet, ICMP, BGP, dentre outros.

Domain Name System (DNS)

Vimos até este ponto os protocolos utilizados na internet para encaminhamento e roteamento de pacotes, bem como os protocolos de transferência de dados. Serviços que utilizam a internet usam tais protocolos para se comunicarem. Um destes serviços é o Domain Name System (DNS), responsável pela tradução dos endereços IP em nomes, bem como os nomes em endereços IP.

Em um computador local, este processo de tradução pode ser feito através de arquivos que contenham os endereços numéricos e os nomes. É o caso do arquivo hosts presente em sistemas operacionais como o Microsoft Windows e o Linux. Porém, em uma rede com muitos computadores conectados, tanto de forma local quanto remota, usar um arquivo como esse seria inviável, pois é impensável colocar 4 bilhões de endereços num arquivo texto com seus respectivos nomes, além de ficar atualizando, caso os nomes mudem. Neste caso, o DNS será a solução para seu problema.

World Wide Web (Web)

Outro serviço muito utilizado na internet é o World Wide Web (ou simplesmente Web). Ele é fundamental para quem está desenvolvendo aplicações ou sites. Seu principal protocolo é o HyperText Transfer Protocol (HTTP) e sua arquitetura é baseada no paradigma Cliente-Servidor, onde o cliente é chamado de Navegador Web (Browser) e o servidor de Servidor Web.

Os Servidores Web, normalmente, utilizam a porta TCP 80, porém, isso é uma convenção, pois é possível que este serviço possa usar outras portas, como a 8080. Neste caso, no endereço do servidor é preciso deixar explícito o número da porta configurada. Por exemplo: http://www.teste.com.br:3000.

Note que o endereço usado no exemplo anterior usa o formato de um Universal Resource Locator (URL). Este formato permite identificar qual protocolo está sendo usado para comunicação com o serviço Web, o nome da máquina servidora e a porta de acesso. Caso seja a porta padrão da Web, 80, não é preciso explicitar.

 O protocolo HTTP

O protocolo padrão utilizado pela Web é o HyperText Transfer Protocol (HTTP). Ele utiliza um conjunto de métodos ou verbs que definem a comunicação entre um Cliente Web e um Servidor Web. Podemos ver um pouco mais sobre este métodos na página Métodos de requisição do HTTP e a Métodos de requisição HTTP. O HTTP está na versão 2.0, que possui várias vantagens em relação às versões anteriores. Até a versão 1.2, o protocolo usava mensagem em formato textual, podendo ser facilmente lidas por qualquer programa que estivesse monitorando o tráfego de uma rede. No HTTP/2, as mensagens são em formato binário. As versões anteriores ao HTTP/2 usavam técnicas de compressão de dados para transmissão. Estas técnicas de compressão foram incorporadas ao HTTP/2. Assim, protocolos como o SPDY da Google não são mais necessários no HTTP/2.

Além do protocolo HTTP, a Web utiliza um padrão de formatação de páginas chamado Hypertext Markup Language (HTML). Este padrão se encontra hoje na versão 5. Além do HTML, padrões de imagens, vídeos e áudio são usados na Web, atualmente.

Os navegadores Web são responsáveis por renderizar as informações textuais do HTML em informações visuais. Estes navegadores também possuem interpretadores para a linguagem Javascript, usada em páginas Web, bem como o Document Object Model (DOM), que mapeia os elementos HTML para objetos que podem ser manipulados por linguagens como o Javascript ou por tecnologias de estilização de páginas, como o Cascading Style Sheet (CSS).
